Приложение "простое как пять копеек". Есть текст и 2 тыкалки. При нажатии js
отправляет http post запрос в небольшой REST api, спринг форвардит его в наш
бек для обработки, бэк с минимальными изменениями делает транзакцию в postgres,
через спринговый ORM (@Service, @Repository и @Entity), и изменение форвардится
обратно во фронт.

Демо работа корсов (шпионская страница). Запускаешь docker compose up spy (он
сам поднимает основное приложение -- app как зависимость). Spy это тот же самый
контейнер, что и основное приложение, но проброшенный на порт 8081, плюс с его
фронта запросы к апишке идут в основной контейнер, развёрнутый на 8080. Из-за
чего нарушается same-origin rule, и в девтулах браузера видно что эти запросы
падают с ошибкой "CORS Missing Allow Origin".
